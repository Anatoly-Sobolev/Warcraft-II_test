# User Story Map

Документ связывает продуктовую ценность, демо и разработку. Он нужен, чтобы каждый спринт показывал не набор внутренних модулей, а понятный пользовательский прогресс.

## Пользователь

Основной пользователь - игрок, который хочет запустить одиночную RTS-миссию в духе Warcraft II на слабом устройстве и управлять базой, войсками и целями миссии.

## Backbone

| Этап | Что хочет игрок | Критерий готовности |
| --- | --- | --- |
| Запустить игру | Открыть приложение и попасть в меню. | Меню запускается и не требует внешних действий. |
| Выбрать режим | Выбрать кампанию, миссию или skirmish. | Выбор приводит к загрузке матча. |
| Понять задачу | Увидеть цель миссии и стартовые условия. | Цель отображается в HUD или сценарном окне. |
| Управлять картой | Двигать камеру, видеть карту, миникарту и туман. | Игрок ориентируется в игровом поле. |
| Выбирать объекты | Выбрать юнитов и здания. | HUD показывает выбранные объекты и доступные команды. |
| Добывать ресурсы | Отправить рабочего на золото/лес/нефть. | Ресурсы растут по правилам Simulation. |
| Строить базу | Построить базовые здания. | Ресурсы списываются, здание появляется и достраивается. |
| Производить армию | Заказать юнитов и улучшения. | Очередь работает, юнит появляется после времени производства. |
| Вести бой | Отдать приказ атаки и победить врага. | Урон, смерть, цели и события считаются в Simulation. |
| Выполнить миссию | Достичь цели победы или получить поражение. | Result screen отражает итог матча. |
| Продолжить прогресс | Перейти к следующей миссии или сохранить состояние. | Прогресс сохраняется и восстанавливается. |

## Вертикальные срезы

| Срез | Описание | Минимальная ценность |
| --- | --- | --- |
| VS-01: Запуск и навигация | Меню, выбор режима, старт матча. | Проверяющий может открыть проект и дойти до игры. |
| VS-02: Базовое управление | Камера, выбор, HUD, команды движения/атаки. | Игрок управляет юнитами. |
| VS-03: Экономика | Рабочий добывает ресурсы, UI показывает изменения. | Появляется RTS-цикл развития. |
| VS-04: Строительство | Рабочий строит здания, занятость карты учитывается. | Игрок расширяет базу. |
| VS-05: Производство | Здания тренируют юнитов и исследуют улучшения. | Игрок готовит армию. |
| VS-06: Бой | Юниты атакуют, получают урон и умирают. | Есть конфликт и тактическая цель. |
| VS-07: Сценарии | Миссии, цели, триггеры, победа/поражение. | Демо становится кампанией, а не песочницей. |
| VS-08: Сохранение и производительность | Save/load, профилирование, слабое железо. | Проект готов к длинной игре и целевой платформе. |

План спринтов можно менять, но отчет каждого спринта должен ссылаться на конкретные user stories из этой таблицы.
