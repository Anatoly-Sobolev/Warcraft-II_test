# Ручная проверка Markdown-ссылок

Скрипт пока не добавляем: на старте достаточно ручной процедуры, чтобы не писать
вспомогательный код раньше времени.

## Когда проверять

- после переноса документов;
- перед пятничной сдачей;
- перед merge задачи, которая меняет README или `docs/`;
- после переименования файлов.

## Что проверять

1. В `README.md` открыть ссылки на запуск, архитектуру, тесты и отчеты.
2. В `warcraft-ii/docs/README.md` открыть все ссылки из таблиц.
3. В актуальном sprint report открыть ссылки на user stories, тесты и механику.
4. Проверить, что нет пустых ссылок вида `[]()` или `[текст]()`.
5. Проверить, что нет ссылок на старые места файлов после переноса.

## Минимальный результат

В sprint report записать:

```md
- README links: pass/fail/not run
- docs index links: pass/fail/not run
- sprint report links: pass/fail/not run
- Комментарий:
```

Если ссылка не открывается, это P0/P1-дефект для пятничной сдачи, потому что
проверяющий может не добраться до демо или нужного артефакта.
