# Кампании — соглашение об именах

Каждая кампания — это отдельная папка с одним `campaign.tres` и набором миссий.
Сейчас в каждой папке лежит только `mission_01` как образец. Остальные миссии
добавляются по тому же шаблону копированием — это наполнение данными, а не новый код.

## Структура одной кампании

```
<campaign>/
├── campaign.tres              # порядок и условия открытия миссий
├── mission_01.tres            # описание миссии: цели, триггеры, диалоги
├── ...                        # mission_02.tres, mission_03.tres, ...
└── maps/
    ├── mission_01_logic.tres  # логическая карта (массивы клеток для Simulation)
    ├── mission_01_visual.tres # визуальная карта (тайлсет, декор для Presentation)
    └── ...
```

## Сколько миссий

| Кампания               | Миссии        |
|------------------------|---------------|
| `base_alliance`        | mission_01 … mission_14 |
| `base_horde`           | mission_01 … mission_14 |
| `expansion_alliance`   | mission_01 … mission_12 |
| `expansion_horde`      | mission_01 … mission_12 |

## Почему карта разделена на logic и visual

`Simulation` не должна загружать тайлсеты, декор и текстуры. Логическая карта —
это компактные массивы (проходимость, ресурсы, стартовые позиции), визуальная —
только то, что рисует `Presentation`. Так одна и та же логика карты может
перерисовываться разным графическим профилем без влияния на симуляцию.
