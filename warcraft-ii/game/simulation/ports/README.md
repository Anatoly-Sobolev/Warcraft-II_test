# simulation/ports

> ↑ [simulation](../README.md) · 🏛 [Архитектура](../../../docs/architecture/architecture.md)

**Ответственность:** **единственные двери** между симуляцией и остальным миром.
Снаружи модули общаются с `simulation` только через эти типы — никогда напрямую с
`storage/` или `spatial/`.
**Владелец состояния:** состоянием не владеют — это контракты (вход команд / выход подготовленных данных).

> ⚠️ Это центральный контракт проекта. Изменил тип здесь — обнови README модулей-потребителей (`input`, `ui`, `presentation`, `scenario`).

## Вход (команды)
| Файл | Назначение |
|---|---|
| `command_sink.gd` | Приём `GameCommand`. Очередь наружу не раскрывается. |

## Запросы (read-only)
| Файл | Кто потребитель | Назначение |
|---|---|---|
| `selection_query.gd` | `input` | Доступность и видимость выбора. |
| `command_query.gd` | `input`, `ui` | Возможные действия, цели, размещение здания. |
| `scenario_query.gd` | `scenario` | Узкие проверки условий миссии. |
| `simulation_event_reader.gd` | `scenario`, `presentation` | Неизменяемые пакеты событий по такту. |
| `simulation_ui_query.gd` | `ui` | Данные экономики, производства, технологий. |

## Подготовленные данные
| Файл | Кто потребитель | Назначение |
|---|---|---|
| `selection_view_data.gd` | `ui` | HP/состояние выбранного по списку ID. |
| `simulation_ui_data.gd` | `ui` | Versioned данные для HUD. |
| `render_change_buffer.gd` | `presentation` | Созданные/изменённые/удалённые сущности. |
| `terrain_change_buffer.gd` | `presentation` | Изменённые клетки карты. |
| `fog_change_buffer.gd` | `presentation` | Изменения explored/visible. |
| `minimap_change_buffer.gd` | `presentation` | Компактные изменения миникарты. |

## Инварианты
- Порты не возвращают изменяемые внутренние массивы.
- `ui` не читает сырые `SimulationEvent` — только подготовленные данные.
- События доставляются ровно один раз и в порядке тактов; dirty-буферы можно объединять между кадрами.
