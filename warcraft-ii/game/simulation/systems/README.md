# simulation/systems

> ↑ [simulation](../README.md) · 🏛 [Архитектура](../../../docs/architecture/architecture.md)

**Ответственность:** пакетная обработка сущностей — вся изменяющая логика мира.
Системы читают/пишут `../storage/`, используют `../spatial/` и `../navigation/`.
**Владелец состояния:** состоянием не владеют — изменяют хранилища.

## Порядок выполнения в такте (детерминирован)
Порядок важен: смерть и освобождение ID — в самом конце, чтобы сущность не исчезла
посреди работы другой системы.

```
command → economy → terrain → construction → production → ability →
status_effect → movement → transport → combat → projectile → visibility →
ai* → cleanup
```
(*`ai_system` выбирает стороны для планирования; см. `../ai/`. Частоты систем — в `simulation_config.gd`.)

## Файлы
| Файл | Ответственность |
|---|---|
| `command_system.gd` | Проверка команд, назначение/отмена приказов, разрешение конфликтов. |
| `economy_system.gd` | Добыча, перенос, возврат ресурсов, стоимость, население. |
| `terrain_system.gd` | Истощение леса/ресурсов, изменение клетки и проходимости. |
| `construction_system.gd` | Размещение, стройка, завершение, ремонт. |
| `production_system.gd` | Очереди юнитов, исследований, улучшений. |
| `ability_system.gd` | Проверка маны/целей, запуск способности. |
| `status_effect_system.gd` | Модификаторы, длительность, снятие эффектов. |
| `movement_system.gd` | Следование пути, резервирование, смена клетки. |
| `transport_system.gd` | Погрузка, пассажиры, выгрузка. |
| `combat_system.gd` | Проверка атаки, cooldown, урон/снаряд. |
| `projectile_system.gd` | Перемещение и попадание логических снарядов. |
| `visibility_system.gd` | `explored`/`visible` для каждой стороны. |
| `ai_system.gd` | Выбор сторон для планирования (фаза для `../ai/`). |
| `cleanup_system.gd` | Смерть, освобождение ссылок, клеток и ID. |

## Инварианты
- Системы не создают `Dictionary`, lambda и временные объекты в горячем цикле.
- Изменения проходимости регистрируются единым методом (обновляет occupancy + версии регионов + dirty-буфер).
