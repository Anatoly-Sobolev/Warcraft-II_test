# simulation

> ↑ [game](../README.md) · 🏛 [Архитектура](../../../ARCHITECTURE.md)

**Ответственность:** **авторитетный игровой мир** — единственный источник правды о
матче. Юниты, стороны, экономика, технологии, приказы, движение, бой, способности,
карта, AI, туман войны, номер тика и RNG.
**Владелец состояния:** всё перечисленное. Это самый большой и самый «горячий» модуль.

## Ключевая идея
Юнит — это `EntityId` (64-битное число: индекс + поколение) плюс компоненты в
плотных типизированных массивах (Data-Oriented). Системы пакетно обрабатывают
только тех, у кого есть нужный компонент. Никаких `Node`, `_process()` и спрайтов.

## Контракт
- **Принимает:** только `GameCommand` (через `ports/command_sink`). Игрок, AI и сценарий шлют одинаковые команды.
- **Отдаёт:** через `ports/` — события, dirty-буферы (render/terrain/fog/minimap), ViewData. Внутренние массивы и сетки наружу **не** отдаёт.
- **Запрещено:** обращаться к `Control`, `Sprite2D`, аудио, файловой системе, платформе. Должен работать в тестах без сцены.

## Корневые файлы
| Файл | Назначение |
|---|---|
| `simulation_world.gd` | Хранилища, сетки, сервисы, системы и порты вместе. |
| `simulation_runner.gd` | Накопление времени, номер тика, запуск систем (с лимитом догоняющих тактов). |
| `simulation_config.gd` | Tick rate, интервалы систем, ограничители нагрузки. |
| `simulation_snapshot.gd` | Явное сериализуемое состояние симуляции. |

## Инварианты
- Фиксированный такт; порядок систем детерминирован (см. `systems/README.md`).
- Долговременная ссылка хранит полный `EntityId` (с поколением), а не индекс.
- Любое изменение статической проходимости делает кэш пути устаревшим (см. `spatial/` + `navigation/`).
