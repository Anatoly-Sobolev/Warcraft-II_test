# simulation/storage

> ↑ [simulation](../README.md) · 🏛 [Архитектура](../../../../ARCHITECTURE.md)

**Ответственность:** компонентные данные сущностей в плотных типизированных массивах
(Structure of Arrays). **Один файл = одно назначение данных.** Общие данные (позиция,
владелец, здоровье) не дублируются между типами сущностей.
**Владелец состояния:** все компоненты сущностей матча.

## Как читать/писать
Системы (`../systems/`) перебирают только активные индексы через `DenseIdSet`.
Снаружи модуля хранилища напрямую недоступны — только через `../ports/`.

## Файлы
| Файл | Основные данные |
|---|---|
| `identity_storage.gd` | `DefinitionId`, тип представления, базовые флаги. |
| `transform_storage.gd` | позиция, направление, слой движения, высота. |
| `ownership_storage.gd` | `side_id`, команда, принадлежность миру. |
| `health_storage.gd` | текущее/макс. HP, броня, состояние смерти. |
| `vision_storage.gd` | радиус обзора, активность источника. |
| `movement_storage.gd` | скорость, путь, сегмент, локальная цель, застревание. |
| `order_storage.gd` | тип приказа, цель, позиция, фаза, флаги. |
| `combat_storage.gd` | профиль атаки, cooldown, дальность, цель. |
| `worker_storage.gd` | переносимый ресурс, добыча, стройка, ремонт. |
| `building_storage.gd` | footprint, прогресс, rally point, состояние. |
| `resource_node_storage.gd` | тип и остаток шахты/нефти. |
| `terrain_runtime_storage.gd` | изменённые клетки, древесина в лесу. |
| `production_storage.gd` | очереди юнитов/исследований, время. |
| `ability_runtime_storage.gd` | мана, cooldown, charges. |
| `status_effect_storage.gd` | источник, цель, тип, такты, стеки. |
| `projectile_storage.gd` | источник, цель, профиль полёта, payload. |
| `transport_storage.gd` | вместимость, пассажиры, точки выгрузки. |
| `side_storage.gd` | до 8 сторон: контроллер, фракция, цвет, ресурсы, население, технологии. |
| `statistics_storage.gd` | добыча, производство, потери, статистика миссии. |

## Инварианты
- Изменяемые планы AI здесь не лежат — они в `../ai/ai_state_storage.gd`.
- Лес — не сущность, а состояние клеток в `terrain_runtime_storage`.
