# simulation/spatial

> ↑ [simulation](../README.md) · 🏛 [Архитектура](../../../docs/architecture/architecture.md)

**Ответственность:** пространственные структуры карты — поверхность, проходимость,
занятость, резервирование, видимость и версии проходимости для проверки кэша путей.
**Владелец состояния:** сетки карты и видимости.

## Файлы
| Файл | Назначение |
|---|---|
| `map_grid.gd` | Базовая поверхность, стоимость, допустимые слои движения. |
| `static_occupancy_grid.gd` | Здания, стены, постоянные препятствия. |
| `dynamic_occupancy_grid.gd` | Фактически занятые клетки. |
| `reservation_grid.gd` | Намерение занять клетку на следующем шаге. |
| `spatial_index.gd` | Быстрый поиск соседей/целей в области. |
| `visibility_grid.gd` | `explored`/`visible` по каждой активной стороне. |
| `path_region_versions.gd` | Версии статической проходимости областей карты. |

## Инварианты
- `PathRegionVersions` растёт при **любом** изменении статической проходимости
  (вырубка дерева, появление/снос здания/стены, смена поверхности).
- Динамические юниты проходимость **не** меняют (учитываются через occupancy/reservation).
- Только `simulation` читает эти сетки. `presentation` получает dirty-буферы из `../ports/`, а не сетки.
