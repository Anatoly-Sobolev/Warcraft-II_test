# simulation/core

> ↑ [simulation](../README.md) · 🏛 [Архитектура](../../../../ARCHITECTURE.md)

**Ответственность:** фундамент симуляции — идентичность сущностей, очередь команд,
события, случайность, расписание систем.
**Владелец состояния:** реестр сущностей и поколений, очередь команд, буфер событий, состояние RNG.

## Файлы
| Файл | Назначение |
|---|---|
| `entity_id.gd` | Кодирование `EntityId` = generation(32) + index(32). |
| `entity_registry.gd` | Создание/освобождение ID с проверкой поколения. |
| `dense_id_set.gd` | Плотный перебор активных компонентов. |
| `entity_factory.gd` | Создание сущности из `Definition` и заполнение хранилищ. |
| `game_command.gd` | Одноразовое намерение игрока/AI/сценария. |
| `command_queue.gd` | Детерминированная сортировка и выдача команд целевого такта. |
| `unit_order.gd` | Длительное состояние исполнения принятой команды. |
| `simulation_event.gd` | Завершившийся факт симуляции. |
| `event_buffer.gd` | Буфер событий по тактам (доставка ровно один раз). |
| `simulation_random.gd` | Единственный сохраняемый источник случайности. |
| `tick_scheduler.gd` | Какие редкие системы выполнять в текущем такте. |

## Инварианты
- `EntityId == 0` — отсутствие сущности.
- Конфликт команд одной сущности решается: `source_priority → sequence → insertion_order`.
