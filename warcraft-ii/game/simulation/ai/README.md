# simulation/ai

> ↑ [simulation](../README.md) · 🏛 [Архитектура](../../../../ARCHITECTURE.md)

**Ответственность:** планирование сторон под управлением компьютера (до 7).
AI создаёт **те же `GameCommand`, что и игрок** — не меняет мир напрямую и не читает
скрытые данные, если это не разрешено профилем сложности.
**Владелец состояния:** планы и решения AI (`ai_state_storage`).

## Файлы
| Файл | Назначение |
|---|---|
| `ai_controller.gd` | Планирование одной стороны. |
| `ai_state_storage.gd` | Планы, cooldown решений, известные цели, директивы. |
| `economy_planner.gd` | Рабочие, ресурсы, стройка, производство. |
| `army_planner.gd` | Защита, группы, атаки. |
| `ai_directive.gd` | Ограничение от сценария: `DEFEND_AREA`, `ATTACK_TARGET`, `PASSIVE_UNTIL_TRIGGER`… |

## Инварианты
- Какие стороны планируют в текущем такте, выбирает `../systems/ai_system.gd` (распределение нагрузки).
- AI выводит команды через `../ports/command_sink` — отдельного пути в мир у него нет.
